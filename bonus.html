Jshell

Jshell is a command line tool that allows you to interact with Java easily and quickly, like using the command node to open up a node console or using the python3 command to interact with python3 in the terminal. Now this also works for Java! This is surprising because unlike python3 or Node.js, Java requires compilation. You can reference the official site for more info.

Make sure you have JDK 9 or above installed. Now open up your command line, and type jshell to enter jshell. If you don't know what a command line or a terminal is, then I suggest you to do some research on your own first.

Once you're in, you can try some cool stuff. For instance, declare an array, and jshell will actually display the data in the array in the next line. Or you can try some arithmetics, jshell will act like a calculator and return the result. (Although to be honest, Java is not good with math)

Nevertheless, we only bring you with the info, the most important thing is explore it yourself!

Bitwise Operators

A cool hack

Question, what will be the output of the following line?

System.out.println(123&456<<2);

You must be very confused if I tell you that the output will be the number 32. Next we'll see why that is the case, and unlock a cool hack for you.

List of Java bitwise operators

Name Symbol Description
Bitwise AND & Perform AND operations to operands in binary form bit by bit.
Bitwise OR | Perform OR operations to operands in binary form bit by bit.
Bitwise XOR ^ Perform XOR operations to operands in binary form bit by bit.
Bitwise complement ~ Flip the bits of a single operand.
Left shift << Shift a number in binary form to the left, the empty spot at the rightmost digit is filled with 0.
Right shift >> Shift a number in binary form to the left, the empty spot at the leftmost digit is filled with the original digit.
Zero fill right shift >>> Shift a number in binary form to the right, the empty spot at the leftmost digit is filled with 0.

Bitwise operators in action

What does all that mean? Let's see them in action. We all know what an AND is, logically equivalent to short circuit AND boolean operator && in Java. Where as && can only operate on booleans, bitwise operator & can also act on integers.

In order to perform the operation, we have to first convert numbers into binary form, for instance the number 123 and 456 are 1111011 and 111001000 in binary respectively. But we cannot start our operations until binary numbers are turned to their representations in computer format. Negative numbers will not be dealt here, because they involve a whole other topic. For positive numbers, we add a 0 at the front.

In the case of 123&456<<2, the left shift operator is calculated before the bitwise AND. So 456<<2 is 011100100000. Now we can calculate the entire expression. We can regard digit 1 in a binary number as true in a boolean expression, and the logic follows that only two 1s can get us a 1(just like only two trues can return a true with &&). Now let's line up the numbers (for a positive binary number we simply add 0s to the front) and perform a bit-by-bit AND operation!

123 000001111011 456<<2 & 011100100000 -------------------------- RESULT 000000100000 Decimal: 32

I'm sure you're smart enough to figure out the rest with this demo seen, give them a shot and try them in jshell!

Usages of bitwise operators

How can such things be useful, you might ask. In fact, shifts can perform efficient division or multiplication by 2 because they resemble the states of computers at their most fundamental levels. In addition to this most common usage, it's also a great thing to show off with (haha). I'm sure your friend would be puzzled when he or she saw you using i = Math.abs(~i) instead of i++.
